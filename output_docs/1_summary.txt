In iTextSharp 5.5.3 the `Font` property of **PdfPTable** is just a shortcut for setting the font of the table’s **DefaultCell**.  
When you set

```csharp
PdfPTable table = new PdfPTable(2);
table.Font = new Font(Font.FontFamily.HELVETICA, 10, Font.NORMAL);
```

the default cell that is used by `AddCell(string)`, `AddCell(Phrase)`, etc., will now use that font.  
The property does **not** go through the cells that are already in the table and change their fonts.  
So:

* `Font` only affects cells that are added **after** the property is set, or cells that use the default cell.  
* Cells that were already added keep whatever font they were created with.

If you need every cell to use a particular font, either set the default cell’s font **before** adding cells, or iterate over the table’s cells and set each cell’s font individually.
